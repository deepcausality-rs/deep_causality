## 5. Stage 2c/3 — Solve engine (`tensor-train-solve`)

- [x] 5.1 Add `SolveConfig<T>` (`solve_config/mod.rs`: `max_sweeps`, `tol`, `ridge`) and extend `CausalTensorError` with `SweepDidNotConverge` (+ `Display`)
- [x] 5.2 Implement the shared one-site ALS sweep machinery (`solve/local.rs`): forward-then-back site order, ridge-regularized local solves, `solve_dense` Gaussian solver, deterministic random init, per-bond rank caps
- [x] 5.3 Implement `solve::fit` (TT completion — block-diagonal-over-physical-index local LLS from samples) and `solve::linear` (`A x = b` via the normal-equation operator `G = AᵀA`, rhs `c = Aᵀb`, one-site ALS with running `<x|G|x>` / `<x|c>` environments)
- [x] 5.4 Implement `integrate` on the `TensorTrain` trait (per-site weight contraction; total on the order-0 identity)
- [x] 5.5 `solve::eigen` (**DMRG3S** ground state — single-site DMRG sweep solving each site's local eigenproblem in mixed-canonical gauge via a `Scalar`-bound cyclic-Jacobi `sym_eig`, plus AMEn projected-residual subspace expansion to grow the bond; reuses the shared `build_local_h`/`env_g_*` machinery) — **Stage 3**. Ref: Hubig–McCulloch–Schollwöck–Wolf 2015, *Phys. Rev. B* 91, 155115 (arXiv:1501.05504). Tests recover a planted symmetric ground state (λ + eigenvector + residual) at f32/f64/Float106; not-square error path. Also hardened `TensorTrain::norm` to clamp `⟨x|x⟩≥0` (a near-zero train otherwise `sqrt`s a tiny negative → `NaN`).
- [x] 5.6 `solve::tdvp_step` — **two-site TDVP2** (single forward sweep; forward two-site `exp(+dt·H₂)`, truncated-SVD split, backward single-site `exp(−dt·H₁)`; rank-adaptive via the split, norm-conserving for a skew generator). Real scaling-and-squaring matrix exponential (`expm_scaled`), `Scalar`-bound. Refs: Haegeman et al. 2016 (*Phys. Rev. B* 94, 165116); Paeckel et al. 2019 (*Ann. Phys.* 411, 167998). Tests: norm conservation under a skew operator (f32/f64/Float106), exact match to dense `exp(M·dt)·x` on a 2-site block (f64), shape-error paths. **CBE bond expansion deferred** (the optional frontier; plain two-site already grows rank).
- [x] 5.7 Re-export `solve` module + `SolveConfig` from `lib.rs`
- [x] 5.8 Tests (f32/f64/Float106 where applicable): `integrate` vs dense sum + error path; `fit` recovers a known train from full samples (f64/Float106); `linear` recovers `x` with `A·x ≈ b` (f64/Float106); config errors. clippy clean, workspace builds. (Caught a real conditioning point: the ridge floors achievable accuracy, so it must be precision-scaled for the tighter `Float106` target.)

## 6. Stage 4 — Scalar generality (`tensor-train-scalars`)

- [x] 6.1 Generalize the SVD/QR/inner kernels to the complex case (Hermitian transpose, real singular values, `⟨a|b⟩ = Σ aᵢ* bᵢ`) via conjugation. **Done** via the `deep_causality_num::ConjugateScalar` bridge trait (sibling `NormedScalar`): Hermitian one-sided Jacobi SVD (complex Givens `c·x − conj(ρ)·s·y`, real σ in `T::Real`, `Vᴴ`), complex Householder QR, Hermitian `inner` (`Σ conj·`), real-clamped `norm`. All reduce exactly to the real algorithms. `Truncation<T::Real>` threads the real magnitude.
- [x] 6.2 Instantiate and test the stack at `Complex<f64>`: **state trains** (TT-SVD round-trip, conjugated inner with real `⟨x|x⟩`, real norm, complex add/scale, lossless round + isometric canonical cores) and **MPO operators** (operator TT-SVD round-trip, `apply` = MPO·MPS vs dense matvec, `compose` equivalence). **Now also complex:** `cross` (modulus pivoting, real residual), and the solvers **`linear`/`fit`/`tdvp`** (`SolveConfig<T::Real>`, modulus-based `solve_dense`/`expm_scaled`, real residuals; the `env_g`/`env_c` contractions conjugate the bra so the effective operator is genuinely Hermitian — identity for reals, a latent fix). Complex TDVP conserves norm under a skew-Hermitian generator; complex AMEn recovers `x` from `A·x=b`; complex ALS fit recovers a known train. Tests: `op_tensor_complex_tests`, `causal_tensor_train_complex_tests`, `causal_tensor_train_operator_complex_tests`, `causal_tensor_train_cross_complex_tests`, `causal_tensor_train_solve_complex_tests`. **`eigen` is now complex too** — `sym_eig` was rewritten as a **complex Hermitian** Jacobi (`Uᴴ A U` with complex Givens, reducing to the real symmetric Jacobi for real `T`) and `symmetrize` is now Hermitian; `solve::eigen` recovers a planted complex Hermitian ground state (DMRG3S). **The whole stack now instantiates at `Complex<f64>`** (kernels, states, operators, cross, linear/fit/tdvp/eigen). Only `apply_nonlinear` stays `Scalar` (it needs `T::nan`).
- [x] 6.3 Confirm the algebra/AD path is bound on `Scalar` (not `Field`); instantiate and test at `Dual<f64>` for forward-mode AD (gradient vs central finite differences to `√ε`; truncation decisions on the real part). **Done** — `Dual<f64>` satisfies `Scalar` (it impls `Real + Div + FromPrimitive`), so the whole stack differentiates with **zero library changes**. Tests in `causal_tensor_train_scalar_tests.rs`: AD of `norm` (via `from_cores`) and AD *through the TT-SVD kernel* (via `from_dense` + `eval`), both vs central finite differences.
- [~] 6.4 Test the mixed `Dual<Complex<f64>>` case — **descoped.** `Dual<T>` is `ConjugateScalar` only when `T: Scalar`, and `Complex` is not `Scalar` (ℂ is unordered), so `Dual<Complex>` is not a `ConjugateScalar` without an additional nested impl. AD-through-complex is an exotic combination with no current consumer; recorded as future work, not built (would belong to a follow-on change).
- [x] 6.5 The full precision/scalar matrix is covered: real generic bodies run `f32`/`f64`/`Float106`; `Dual<f64>` (forward-mode AD) and `Complex<f64>` (Hermitian stack) are each exercised at every layer (kernels, states, operators, cross, solvers) via dedicated `*_complex_tests.rs` / `*_scalar_tests.rs` files rather than one unified generic body.
