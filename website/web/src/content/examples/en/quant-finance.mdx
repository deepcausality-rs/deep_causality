## The problem

A naive trading rule fires on a single condition, ignores the regime around it, and produces too many false positives to trust. A more honest formulation chains conditions: the fast/slow moving-average cross *and* a volume confirmation *and* a regime gate. Each condition is a unit; the trade only fires when all units agree on the same window.

## The shape of the code

Each rule is a `Causaloid`. Composition (`&`) returns a new Causaloid whose evaluation is the conjunction of its parts. The composed Causaloid runs against a `SlidingWindow` of recent ticks.

```rust
use deep_causality::*;
use deep_causality_data_structures::SlidingWindow;

#[derive(Clone)]
struct Tick {
    price: f64,
    fast_ma: f64,
    slow_ma: f64,
    volume: f64,
    median_volume: f64,
}

let cross = Causaloid::new(|t: &Tick| Ok(t.fast_ma > t.slow_ma));
let confirm = Causaloid::new(|t: &Tick| Ok(t.volume > t.median_volume * 1.4));
let signal = cross & confirm;

let mut window: SlidingWindow<Tick, 64> = SlidingWindow::new();
for tick in feed.iter() {
    window.push(tick.clone());
    if signal.reason_with(window.latest())?.is_true() {
        engine.long(ticker, qty)?;
    }
}
```

## Walkthrough

The two `Causaloid::new` constructors wrap pure closures. They take a `Tick` reference and return a `Result<bool, _>`. Wrapping them as Causaloids gives them an algebra: conjunction (`&`), disjunction (`|`), negation (`!`), and the ability to compose into larger structures without losing the per-unit evaluation contract.

`reason_with` is the entry point. It takes a value and returns a `CausalDecision` carrying the boolean outcome plus a record of which sub-rules fired. That record is what makes the chain auditable when an order does or does not go out.

`SlidingWindow<T, N>` is a zero-allocation ring buffer. The const generic `N` fixes the window size at compile time, so there is no surprise heap traffic on the hot path.

## Run it

```bash
cargo new --bin trading_signal
cd trading_signal
cargo add deep_causality deep_causality_data_structures
# paste the example into src/main.rs and feed it your tick source
cargo run --release
```

## Why this is a good fit

The conditions change. The thresholds change. The number of stages grows over a season. Pulling each stage out as a Causaloid lets you swap, audit, and test one stage without touching the others.
