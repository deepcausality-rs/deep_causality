[Walk 1](/tutorial/stage-1-corridor/) flew one descent and forked a paused
state to choose a command. This walk flies the same descent across six
atmospheres and reduces the result to a table with error bars.

The counterfactual axis has moved. Walk 1 held the world fixed and varied a
command. Here we hold the command fixed and vary the world. The DSL has a
separate vocabulary for that, and this walk is mostly about why.

## Before we start

This one takes about five minutes, so start it now and read while it runs.

```bash
cargo run --release -p avionics_examples --example plasma_blackout_weather
```

It flies 48 full coupled descents concurrently. Your output ends with a table
and eight gates.

## Station 1: The entire campaign is one expression

Open `weather/main.rs`. Past the setup, the whole program is this:

```rust
let table = CfdFlow::study("weather-dispersion table")
    .save_log(audit_dir.join("weather.audit"))
    .cases(model::weather_cases())
    .baseline(model::standard_day)
    .alternate(model::weather_world)
    .ensemble(constants::MC_DRAWS)
    .couple(|case, draw| world::corridor_coupling(model::bias_departure(case.d_temp), draw))
    .march_for(constants::STEPS, world::initial_field)
    .reduce_ensemble(model::world_row)
    .inspect(utils_print::print_rows)
    .record(&table_path)
    .gates(model::weather_gates())
    .verdict()?;
```

Forty-eight coupled descents, a reduction, a recorded artifact, and eight
acceptance criteria. Read it top to bottom and it says what it does.

**Why this shape.** The phases are ordered by a type-state, so the compiler
rejects a campaign assembled in the wrong order. You cannot reduce before you
march, and you cannot gate before you reduce. The crate pins this with
compile-fail doctests, which means the ordering is a property of the API
rather than a convention in a style guide.

## Station 2: `baseline` and `alternate`, and the claim they make structural

The two lines that matter most in this program:

```rust
.baseline(model::standard_day)     // the validated origin, built once
.alternate(model::weather_world)   // six counterfactual atmospheres, each marked
```

Open `model.rs` and look at what they call. Both `standard_day` and
`weather_world` funnel into the same private `world_cfg` builder; the
baseline just passes a zero temperature offset and a unit density scale.

**Why it is built this way.** Consider the alternative: six independent
`CompressibleMarchConfig` values, one per condition, each constructed
separately. That is how this activity is normally done, and it has a specific
weakness. Every report that comes out of such a campaign contains the sentence
"everything else was identical between runs," and that sentence is a *claim by
the author*. Nothing enforces it. A drifted constant in one of the six
constructors is invisible.

With `.baseline(..).alternate(..)`, the six worlds are the same validated
description with one declared difference applied. "Everything else was
identical" stops being a claim and becomes a property of the program.

**And it is stamped.** Every alternated world carries a
`!!ContextAlternation!!` marker into its provenance log, naming the baseline
it departs from. Gate 1 checks this:

```text
[PASS] (1) counterfactual audit trail: every dispersion world carries the
       !!ContextAlternation!! marker naming its baseline
```

So the table's provenance is not a folder of filenames whose meaning lives in
someone's memory. Each row can name what it is a counterfactual *of*.

## Station 3: `ensemble`, and the diagnosis it once rescued

```rust
.ensemble(constants::MC_DRAWS)   // 8 deterministic receiver-noise draws
```

Open `constants.rs` and read the comment above `MC_DRAWS`:

```rust
/// Receiver-noise realizations flown per weather condition. Each draw is a deterministic
/// phase-shifted low-discrepancy sequence (no RNG dependency; draw 0 is the corridor's
/// sequence), so the whole campaign remains bit-reproducible while the drift cells gain error
/// bars.
pub const MC_DRAWS: usize = 8;
```

**Why deterministic draws rather than an RNG.** A campaign seeded from an RNG
gives different numbers on every run, so a gate on its output has to be loose
enough to absorb the scatter, which means it is too loose to catch a real
regression. Deterministic low-discrepancy draws give a genuine spread across
the ensemble while keeping the whole campaign bit-reproducible. You get error
bars *and* a gate that bites.

**Why the ensemble exists at all.** This is the part worth remembering,
because it was learned the hard way. An earlier single-draw version of this
table showed the cold-day row landing 8 percent below its predicted value.
That looks like a physics finding. It was not: when the Monte Carlo mean
arrived, the deviation turned out to be one lucky draw.

A single run per condition cannot tell a mechanism from a draw. That is the
whole argument for the `.ensemble` phase, and it is why gate 4b exists at
station 5.

## Station 4: The coupling closure, and where the draw index goes

```rust
.couple(|case, draw| world::corridor_coupling(model::bias_departure(case.d_temp), draw))
```

This closure receives both the case and the draw index, and it passes both
*into* the physics stack.

**Why that matters.** The draw index is not a wrapper around the simulation
that perturbs its outputs afterwards. It reaches `corridor_coupling`, which
hands it to the IMU model, which is inside the coupled loop. The noise is
applied where the sensor is, on the step the sensor is read.

The same is true of the temperature departure. Look at `bias_departure` in
`model.rs`:

```rust
pub fn bias_departure(d_temp: f64) -> f64 {
    1.0 + IMU_THERMAL_COEFF_PER_K * d_temp.abs()
}
```

The day's temperature scales the accelerometer bias the vehicle actually
flies, while the navigation filter keeps its standard-day priors in every
world. The mismatch between the instrument flown and the instrument assumed
is the phenomenon this table exists to measure, and it is expressed as a
parameter threading into the stack, not as a post-processing correction.

<p class="callout-title">Notice</p>
<p>In your finished table, the onset and dwell columns carry no error bar
while the drift column does. That is not an oversight. Receiver noise never
touches the flow, the chemistry, or the truth trajectory, so those columns are
draw-invariant by construction. An error bar there would imply a scatter that
cannot exist.</p>
</div>

## Station 5: Why the error bars are load-bearing

Look at the two gates at the centre of the run:

```text
[PASS] (4) the INS does not behave as assumed in the cold: polar-winter mean
       blackout drift 68.75 m vs standard-day 45.93 m (1.50x; gate requires 1.2x)
[PASS] (4b) the cold effect is statistically resolved: polar-standard
       separation 22.81 m vs combined sigma 5.69 m (4.0 sigma; gate requires 2)
```

Gate 4 says the effect is large. Gate 4b says the effect is real.

**Why both are needed.** Without 4b, gate 4 compares two means and asserts a
ratio. If the underlying scatter were 20 m, a 22.81 m separation would mean
nothing at all, and the gate would still pass. Gate 4b requires the separation
to clear two combined standard deviations before the finding counts. It
clears four.

This is the difference between a number in a table and a result. The
`.ensemble` phase is what makes the second one available.

**Why `save_log` writes stepwise.** The audit files you watched appear during
the run are flushed as the branches step, not assembled at the end. A campaign
that dies at draw 40 of 48 still leaves the evidence for the 40 that ran.

## Try it yourself

**1. Weaken the statistics.** In `constants.rs`, set `MC_DRAWS` to `2` and
re-run. The means will move a little; the combined sigma will move a lot.
Watch whether gate 4b still clears its 2-sigma requirement. This is the
lucky-draw failure from station 3, reproduced deliberately.

**2. Add a condition.** Find the `WEATHER` table in `constants.rs` and add a
row, say a `very_cold_day` at −55 K and 1.30 density. Nothing else needs to
change: the case axis drives the campaign, the alternation marker is
automatic, and the table gains a row. Gate 3 checks that weather actually
moves the window, so watch the onset spread widen.

**3. Follow one row to its evidence.** Pick any drift cell in the final table.
Open the eight matching files in `weather/audit/` and confirm each names the
baseline it was alternated from. That path, from a published number back to
the runs behind it, is what station 2 was arguing for.

## What this walk established

- **Fork a state** when the branches must share a past (walk 1). **Alternate a
  world** when the branches must share a *description* and differ in one
  declared way.
- Determinism and error bars are not in tension. Deterministic draws give both.
- A table row without an ensemble behind it cannot distinguish a mechanism
  from a draw.
- The campaign's output is an artifact on disk with a provenance trail, not a
  console summary.

That last point is what makes walk 3 possible.
