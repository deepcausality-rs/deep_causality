This is the example the crate exists to make possible, and the one to run first if
you only run one. In about forty seconds it marches a compressible flow, reacts its
plasma chemistry, gates a Kalman filter on the result, forks the running simulation
into seventeen counterfactual worlds, and prints thirteen gate verdicts.

## The problem

A vehicle reenters at Mach 25. The shock layer ionizes, the electron density climbs
past the GPS L1 cutoff, and the vehicle loses GNSS — radio blackout. It flies blind
on inertial navigation until recombination drains the sheath. Then it reacquires,
with whatever navigation error accumulated in the dark.

The guidance question: which bank angle should it commit to, given that the
navigation state it will have at the decision point is degraded by an amount that
depends on how long the blackout lasts, which depends on the flow?

Every part of that sentence is coupled to every other part. That coupling is the
point.

## Blackout is found, not scheduled

The window is not a config parameter. The run marches until the evolved sheath's
electron density crosses the L1 cutoff:

```rust
let onset = CfdFlow::march(&nominal)
    .couple(world::corridor_coupling(1.0, 0))
    .trigger(trigger)
    .from_field(world::initial_field())
    .until(|field, _| field.regime().map(|r| r.gnss_denied).unwrap_or(false))?;
```

```text
[PASS] (1) flow-resolved blackout window: onset at step 119 (74.7 km),
       denied through the peak, exit at 47.0 km after a 58.4 s dwell
```

The chemistry is anchored against flight data:

```text
[PASS] (2) peak n_e vs the RAM-C II anchor: n_e = 3.349e19 m^-3 at the 60.9 km
       passage, in [2.0e18, 5.0e19] around the flight anchor 1e19 m^-3
```

One honest wrinkle the example states itself: exit at 47.0 km sits above the RAM-C II
flight window of 25–30 km. The attribution is the probe's deliberately light
ballistic bundle, not the chemistry — this vehicle decelerates higher. The offset is
ballistics.

## Forking the running simulation

At onset the march pauses. That pause is forked once per candidate bank angle, and
each branch continues from the shared state in its own alternated world.

The fork is O(1) copy-on-write — tensor fields, navigation engine and provenance log
included — and each branch pays for its clone at its first write. Every branch carries
an `!!ContextAlternation!!` marker naming the baseline it came from, so the audit
trail survives the fan-out.

Six coarse branches, then eleven refined:

```text
 0.0deg  20.000 m
 5.0deg  12.812 m
10.0deg   5.807 m
15.0deg   3.120 m   <- committed from the coarse round
20.0deg   9.612 m
40.0deg  22.018 m

fine winner: 13.5deg  2.393 m
```

```text
[PASS] (4e) guidance precision from the sweep: committed 13.5 deg lands 2.39 m
       off the aim vs the ballistic 20.00 m (8.4x better; gate requires 3x)
```

The landscape is not monotone — 40° is worse than doing nothing. That is the useful
part. A sweep that only confirmed "more bank is better" would not have needed to run.

## Navigation through the dark

```text
[PASS] (3) real INS drift -> reacquisition: err 0.3467 m -> 2.5448 m
       dead-reckoning to the peak passage, 0.2775 m after reacquisition
```

A 17-state error-state Kalman filter dead-reckons through the denial window. Error
grows roughly sevenfold in the dark, then collapses below its starting value once
fixes return. The commit decision is made with the degraded number, not the recovered
one, which is the whole reason the flow solve has to be in the loop.

## Precision, again

| Type | Result |
|---|---|
| `f64` | all gates pass in ~35 s |
| `Float106` | identical events, 15–16 significant digits, ~11× wall-clock |
| `f32` | crashes at step 1 |

The `f32` failure is specific and worth knowing: `h²` in the then-flown Saha kernel
evaluates to 4.4e-67, which underflows the `f32` exponent range. Not a precision
problem — a representability one.

## Where this stops being enough

The sheath closure is single-point with `T_e = T_ve` lumped; there is no spatially
resolved reacting layer. Time is compressed, with each coupled step standing for
0.1 s of flight. The marched layer is 2-D because a
[timing study](/boundaries/#timing-3d) put the 3-D fitted marcher 3.6× over the
budget. There are no winds, no aero-coefficient dispersions and no density
perturbations — for those, see the
[weather dispersion table](/examples/plasma-blackout-weather/), which is the
example built to supply them.
