\begin{abstract}
Bornholt et al.'s \texttt{Uncertain<T>} type captures value uncertainty: a quantity exists and only its measurement is noisy. Real workloads also exhibit \emph{presence uncertainty}: clinical reports go missing, sensors drop readings, GPS fixes vanish in tunnels. Folding such absent data into a wide distribution leaks fabricated numbers; wrapping it as \texttt{Option<Uncertain<T>>} collapses a probability into a Boolean and reintroduces the very uncertainty bugs \texttt{Uncertain<T>} eliminates. We present \texttt{MaybeUncertain<T>}, an extension that factors the two axes into a pair of independent random variables, $(\mathit{is\_present}: \Utype\langle \Bool \rangle,\, \mathit{value}: \Utype\langle T \rangle)$. Arithmetic propagates both axes; presence combines under AND. A statistically-gated operator \texttt{lift\_to\_uncertain} converts the type back to plain \texttt{Uncertain<T>} only when a sequential probability ratio test (SPRT) on the presence axis clears a confidence threshold. The implementation reuses Bornholt's sampling machinery, the SPRT logic, and a global sample cache, costing roughly $1.6$ to $2\times$ per sample in benchmark and single-digit microseconds in the worst SPRT case. Three worked examples (clinical trials, intermittent sensors, GPS dropouts) illustrate the extension closing a class of sparse-data bugs that the original abstraction cannot reach.
\end{abstract}
